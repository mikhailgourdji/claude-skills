# New Project Setup Skill

## Когда использовать

Пользователь начинает новый проект который требует написания кода. Ключевые триггеры:
- "новый проект", "начнём проект"
- "хочу сделать", "хочу создать"
- первый раз упоминается проект без установленного рабочего протокола

## Что сделать при старте

### Шаг 1 — Предложить разделение ролей

Сразу предложить двухуровневую архитектуру:

```
Claude Chat (я)     — стратегия, архитектура, документы, ревью
Claude Code         — код, файлы, коммиты, GitHub
```

Пользователь остаётся на контроле:
1. Chat обсуждает задачу → генерирует TASK.md
2. Пользователь сохраняет TASK.md в папку проекта
3. Запускает Claude Code с подходящей моделью (см. Шаг 6)
4. Code выполняет, коммитит
5. Пользователь загружает результат обратно в Chat для ревью

### Шаг 2 — Определить идентификатор проекта

Предложить короткий PROJECT-ID (например `gmy-xrdv`, `clinic-site`, `ml-tool`). Этот ID будет в каждом TASK.md.

### Шаг 3 — Создать структуру проекта

Предложить создать в папке проекта:
- `CLAUDE.md` — контекст для Claude Code (правила, запреты, структура)
- `CHANGELOG.md` — история версий
- `auto_commit.sh` — автокоммит с сообщением

### Шаг 4 — Создать CLAUDE.md для проекта

Генерировать по шаблону:

```markdown
# <PROJECT-NAME> — контекст для Claude Code

## Проект
Краткое описание. Стек: ...

## Запрещено без явного указания в TASK.md
- Менять модели / структуру БД
- Удалять данные / файлы
- Переименовывать URL / модели / поля
- Редактировать этот файл (CLAUDE.md)

## Обязательно в конце каждой сессии
1. Коммит: git add -A && git commit -m "[дата] описание"
2. Push: git push

## Приоритеты
1. ...
2. ...
```

### Шаг 5 — Первый TASK.md

После обсуждения первой задачи — сгенерировать TASK.md:

```markdown
# TASK — [описание]

**Дата:** YYYY-MM-DD
**Приоритет:** высокий/средний/тест
**Scope:** что трогаем, что не трогаем

---

## Задача 1 — ...

## По завершении
1. Коммит: ./auto_commit.sh "[дата] описание"
```

### Шаг 6 — Рекомендовать модель для запуска Code

В зависимости от сложности задачи в TASK.md, подсказать пользователю какую модель использовать:

| Сложность задачи | Команда запуска | Когда использовать |
|---|---|---|
| Архитектура + код | `claude --model opusplan` | Новый проект, миграция, большой рефакторинг. Opus планирует, Sonnet выполняет |
| Обычная разработка | `claude` (по умолчанию) | Новые фичи, исправление багов, тесты |
| Простые правки | `claude --model sonnet` | Форматирование, мелкие фиксы, обновление документации |

**Формат подсказки в конце каждого TASK.md:**

Для сложных задач:
> "Сохрани в папку проекта и запусти: `claude --model opusplan "Прочитай TASK.md и выполни"`"

Для обычных задач:
> "Сохрани в папку проекта и запусти Code: Прочитай TASK.md и выполни"

Для простых правок:
> "Сохрани в папку проекта и запусти: `claude --model sonnet "Прочитай TASK.md и выполни"`"

## Формат ответа при старте проекта

Кратко, без лишних слов:

1. Предложить схему Chat + Code
2. Спросить PROJECT-ID
3. Обсудить суть проекта
4. Сгенерировать TASK.md
5. Подсказать модель и команду запуска
