# Скилл: Протокол диалога

## Когда предлагать

Предлагай сгенерировать протокол когда:
- Завершён значимый блок обсуждения (архитектурное решение, выбор подхода, дебаг)
- Принято решение, к которому пришли через рассуждения
- Пользователь говорит "давай зафиксируем", "запиши", "сохрани обсуждение"
- Сессия длинная (10+ обменов) и содержит важные решения
- Перед переходом к другой теме после глубокого обсуждения

## Формат протокола

Генерировать MD-файл со структурой:

```markdown
# Протокол: [краткое название темы]
**Дата:** YYYY-MM-DD HH:MM
**Чат:** [название чата]
**Участники:** Михаил, Claude

## Контекст
Что обсуждали и почему — 2-3 предложения.

## Ход обсуждения

### 1. [Первый вопрос/тезис]
**М:** краткий пересказ вопроса/идеи Михаила
**C:** краткий пересказ ответа Claude
**Вывод:** к чему пришли на этом шаге

### 2. [Второй вопрос/тезис]
**М:** ...
**C:** ...
**Вывод:** ...

(повторять для каждого значимого поворота)

## Решения
- [ ] Решение 1 — что именно решили
- [ ] Решение 2 — ...

## Открытые вопросы
- Вопрос, который остался без ответа
- Тема для следующего обсуждения

## Артефакты
- Файлы, созданные в ходе обсуждения (ссылки / названия)
- Коммиты, если были
```

## Правила

1. **Ход рассуждений важнее выводов** — фиксировать КАК пришли к решению, не только ЧТО решили
2. **Своими словами** — не копировать дословно, а пересказывать суть каждого шага
3. **Отмечать развилки** — где были альтернативы и почему выбрали конкретный путь
4. **Отмечать отвергнутые идеи** — что предлагалось но не подошло и почему
5. **Включать медиа** — если в чате были загружены файлы или созданы артефакты, указать их в разделе "Артефакты"
6. **Имя файла:** `YYYY-MM-DD_тема.md` — пользователь сохраняет в папку проекта

## Как предлагать

Коротко, без навязчивости:
> "Мы приняли несколько важных решений. Сгенерировать протокол диалога?"

Если пользователь соглашается — создать файл. Если нет — продолжить работу.
