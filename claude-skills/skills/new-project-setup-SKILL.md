# New Project Setup Skill

## Когда использовать

Пользователь начинает новый проект который требует написания кода. Ключевые триггеры:
- "новый проект", "начнём проект"
- "хочу сделать", "хочу создать"
- первый раз упоминается проект без установленного рабочего протокола

## Что сделать при старте

### Шаг 1 — Предложить разделение ролей

Сразу предложить трёхуровневую архитектуру:

```
Claude Chat (я)     — стратегия, архитектура, документы, ревью
Claude Code         — код, файлы, коммиты, GitHub
Cowork              — мост: передаёт задания от Chat к Code,
                      возвращает результаты обратно
```

Объяснить кратко: пользователь остаётся на контроле — читает результат и передаёт его мне. Всё остальное автоматически.

### Шаг 2 — Определить идентификатор проекта

Предложить короткий ID в формате `имя-проекта` (например `gmy-xrdv`, `clinic-site`, `ml-tool`). Этот ID будет в каждом TASK.md чтобы Cowork знал в какую папку копировать.

### Шаг 3 — Настроить Cowork

Если Cowork ещё не настроен — дать инструкции:

1. Создать папку: `mkdir ~/Documents/<PROJECT-ID>-cowork`
2. Скачать CLAUDE.md для Cowork (я генерирую)
3. В Claude Desktop → Cowork → Work in a folder → выбрать папку
4. Проверить: написать Cowork "Прочитай CLAUDE.md и скажи свою роль"

### Шаг 4 — Создать CLAUDE.md для Cowork-папки

Генерировать файл по шаблону:

```markdown
# <PROJECT-NAME> Cowork — контекст

## Роль
Связующее звено: Claude Chat → Claude Code.

## Папки
- ~/Documents/<PROJECT-ID>-cowork/ — эта папка (мост)
- ~/Documents/<PROJECT-FOLDER>/ — папка проекта

## Протокол

### Когда появляется TASK.md:
1. Прочитай поле Project: — убедись что это <PROJECT-ID>
2. Скопируй в ~/Documents/<PROJECT-FOLDER>/TASK.md
3. Запусти Claude Code: cd ~/Documents/<PROJECT-FOLDER> && claude --print "Прочитай TASK.md и выполни" &
4. Сообщи: "Задание передано. Claude Code работает."

### После выполнения (когда Claude Code обновил PROJECT_STATE.md):
1. Прочитай ~/Documents/<PROJECT-FOLDER>/PROJECT_STATE.md
2. Скопируй в LATEST_STATE.md в этой папке
3. Сообщи: "LATEST_STATE.md готов — загрузи его в чат"

## Проекты
- <PROJECT-ID> → ~/Documents/<PROJECT-FOLDER>/

## Правила
- НЕ редактировать файлы проекта напрямую
- НЕ запускать git команды
- Если project неизвестен — спросить
```

### Шаг 5 — Создать структуру проекта

Предложить создать в папке проекта:
- `CLAUDE.md` — контекст для Claude Code
- `PROJECT_STATE.md` — начальное состояние
- `CHANGELOG.md` — история версий
- `auto_commit.sh` — автокоммит

### Шаг 6 — Первый TASK.md

После обсуждения первой задачи — сгенерировать TASK.md с полями:

```markdown
# TASK — [описание]

**Дата:** YYYY-MM-DD
**Project:** <PROJECT-ID>
**Приоритет:** высокий/средний/тест
**Scope:** что трогаем, что не трогаем

---

## Задача 1 — ...

## По завершении
1. Обновить PROJECT_STATE.md
2. ./auto_commit.sh "[дата] описание"
3. Скопировать PROJECT_STATE.md в ~/Documents/<PROJECT-ID>-cowork/LATEST_STATE.md
```

Сказать пользователю: "Положи этот файл в папку `<PROJECT-ID>-cowork/` — Cowork сам передаст его в проект."

## Формат ответа при старте проекта

Кратко, без лишних слов:

1. Предложить трёхуровневую схему (одна схема-картинка или таблица)
2. Спросить ID проекта
3. Проверить настроен ли Cowork — если нет, дать инструкции
4. Перейти к сути проекта
